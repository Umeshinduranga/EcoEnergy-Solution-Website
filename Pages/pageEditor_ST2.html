Home Page

Technical Description

The homepage is structured using semantic HTML5 tags to ensure clarity and functionality. The <nav> tag creates a sticky navigation bar, enhancing user navigation by remaining fixed at the top during scrolling. The <main> tag encapsulates the primary content, providing a clear content hierarchy for both users and search engines. A <section> with a .hero-content class serves as the focal point, housing key text elements like <h2> and <p> to draw attention to the page’s purpose. The <img> tags within .banner-image and .turbine-image classes display a full-width banner and an overlapping turbine graphic, respectively, enhancing visual appeal.

Accessibility

Accessibility is addressed through semantic HTML and CSS choices to support screen readers and keyboard navigation. The <nav> tag clearly defines the navigation area, aiding assistive technologies in identifying interactive elements. Descriptive text in <h2> and <p> within .hero-content ensures content is meaningful out of context. High-contrast colors—like white text (color: white) against a dark background (background-color: rgba(14, 13, 13, 0.8))—improve readability for visually impaired users. The font-size properties (e.g., 300% for h2, 160% for p) ensure text is legible, while line-height: 1.6 on the body enhances text spacing. Links and buttons use :hover and :active states with visible feedback (e.g., color shifts and transforms), aiding users with motor impairments. For images like the banner and turbine, alt text should be added to <img> tags (not shown in CSS but assumed in HTML) to describe their purpose, ensuring inclusivity.

Table Page

Technical Description

The table page employs HTML5 tags like <nav> for consistent navigation and <section class="wrapper"> to encase a data table, built with <table>, <thead>, and <tbody> for structured data presentation. The <header> with <img> and <h1> adds a visual banner, enhanced by CSS position: absolute and animation: popUp for dynamic text entry. CSS styles the .table-container with background-color: #1a1a1a, border-radius: 35px, and box-shadow for a bold, modern look, while thead th uses background-color: #4a2c6f to distinguish headers. Features like .tooltip with position: absolute and tbody tr:hover with transition improve interactivity, justifying a design that balances data clarity with engaging visuals.

Accessibility

Semantic <nav> and <table> with aria-label (e.g., "Table showing renewable energy production trends") aid screen readers in identifying navigation and data structure. High-contrast color: #e0e0e0 text on #2a2a2a table background, paired with large font-size: 20px in .table-container, ensures readability. Tooltips in .tooltip use cursor: help for discoverability, and <img> tags include alt attributes (e.g., "Icon for 2015 data"). The @media (max-width: 768px) rule with overflow-x: auto makes the table scrollable on small screens, enhancing mobile accessibility.

Content Page

Technical Description

The content page uses semantic HTML5 tags like <nav> for a top navigation bar and <div class="container"> to centralize content, ensuring a logical structure for navigation and readability. A fixed <div class="sidebar"> with <a> links provides quick section access, while <h1>, <h2>, and <p> tags organize text hierarchically. CSS applies a dark background-color: #1a1a1a on body and #1b1818 on the container for a sleek look, with position: fixed and padding: 20px 20px 20px 220px accommodating the sidebar. The .image-container uses text-align: center and box-shadow to highlight images, while color properties (e.g., #ffffff for headings) ensure contrast. This design prioritizes accessibility and a modern aesthetic, with a sidebar enhancing navigation efficiency.

Accessibility

Accessibility is enhanced with semantic <nav> and <div class="sidebar"> for clear navigation landmarks, aiding screen readers. The sidebar’s a:focus with outline: 2px solid #cc99ff ensures keyboard navigation support. Images in .image-container include alt attributes (e.g., "Wind Turbine Farm") for screen reader compatibility, while line-height: 1.6 on body enhances text legibility, making the page inclusive.
